=========================================================
AMERICAN DREAM INTERACTIVE SALES DECK - PROJECT MANUAL
=========================================================

1. PURPOSE OF THE PROJECT
---------------------------------------------------------
This project is an interactive, premium "Sales Deck" built as a modern web application for the American Dream Mall. Its primary purpose is to serve as a high-end digital presentation tool to secure brand partnerships, leasing tenants, and major event bookings. 

Rather than presenting potential clients (e.g., luxury brands, event organizers) with a static PDF or PowerPoint presentation, this website provides an immersive, data-rich, and visually stunning experience that highlights the mall's massive scale, luxury avenues, high foot traffic, and unique attractions.

2. TECHNOLOGY STACK
---------------------------------------------------------
The platform is built using a decoupled architecture, splitting the high-performance UI from the AI backend:

Frontend (Web Application):
- React 18: Core UI library for component-based architecture.
- Vite: Lightning-fast build tool and development server.
- TypeScript: Adds strict static typing for robust code.
- Framer Motion: Powers the smooth, premium scroll animations and page transitions.
- React Router DOM: Handles multi-page routing (Home, Leasing, Events, Venues, Sponsors).
- Vanilla CSS: Pure, highly optimized CSS matching a dark-mode, luxury glassmorphism design system.

Backend (RAG Chatbot Service):
- FastAPI: High-performance Python backend framework.
- Uvicorn: ASGI web server for running FastAPI.
- LangChain: The orchestration framework for the Retrieval-Augmented Generation (RAG) architecture.
- ChromaDB: Local persistent vector database used to store and search document chunks.
- Hugging Face Inference API: Cloud-based semantic embedding model (`all-MiniLM-L6-v2`) used to convert text into searchable vectors without requiring local GPU power.
- OpenRouter API: Connecting to the `arcee-ai/trinity-large-preview` LLM to generate intelligent answers with reasoning.

3. CORE FEATURES & MODULES
---------------------------------------------------------
A. Immersive Homepage
   - Dynamic Hero Section with floating statistics.
   - Interactive Scale & Footprint visualizer detailing the 3 Million Sq Ft catchment area.
   - Luxury Showcase (The Avenue) featuring top-tier brands (Rolex, Tiffany & Co, Ferrari) with auto-scrolling preview cards.
   - Multi-layer Dining & Attractions sections highlighting Nickelodeon Universe and DreamWorks Water Park.

B. Deep-Dive Routes (Phase 2 Modules)
   - Leasing Opportunities (/leasing): Detailed views for flagship retail and F&B footprints.
   - Brand Partnerships (/sponsors): Highlighting digital activations and physical co-marketing.
   - Event Bookings (/events): Concerts, galas, and corporate buyout specifications.
   - Venue Directory (/venues): Specific operational data for 'Dream Live' and other massive event spaces.

C. Connect & Lead Generation (CTA)
   - "Take the Next Step" submission forms explicitly routed by commercial interest (Leasing vs. Events).
   - Dynamic hover cards that cross-link directly to Phase 2 deep-dive modules.

4. THE "AD ASSISTANT" AI CHATBOT INTEGRATION
---------------------------------------------------------
To modernize the commercial pitch, a native AI Assistant was integrated to answer granular technical queries from prospective tenants instantly, based exclusively on the mall's internal data.

How It Works (RAG Architecture):
1. Ingestion: A proprietary PDF ("American Dream Mall Research Report.pdf") was processed via a custom `ingest.py` script. The document was split into 500-character chunks, vectorized using the Hugging Face API, and stored permanently in a Chroma vector database on disk.
2. The Backend (/ask Endpoint): When a user asks a question, FastAPI receives the query, converts the question into a vector using Hugging Face, and retrieves the top 3 most relevant textual chunks from ChromaDB. 
3. The LLM Generation: Those 3 chunks, along with the user's exact question, are sent via a strict prompt to OpenRouter's AI processing. The prompt explicitly restricts the AI to *only* answer using the provided chunks. If the answer isn't in the chunks, the AI gracefully declines, preventing hallucinations.
4. The Frontend Window: A React `<Chatbot />` component sits globally in `App.tsx`. It provides a sleek, floating glassmorphism UI in the bottom right corner of the website, maintaining conversation state and displaying loading animations while waiting for the FastAPI server to respond.

5. DEPLOYMENT STRUCTURE
---------------------------------------------------------
The project is built to be deployed individually or combined:
- Frontend: Can be deployed to Vercel, Netlify, or AWS Amplify using standard Node.js build pipelines (`npm run build`).
- Backend: Designed with a production `Dockerfile` exposing port 7860, ready to be immediately deployed into a free Hugging Face Space Docker container as an independent microservice. Using the Hugging Face Endpoint Embeddings guarantees the ingestion and retrieval processes work seamlessly in cloud environments without downloading heavy local models.
